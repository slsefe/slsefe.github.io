AJAX是什么

AJAX 是什么

AJAX, 全称Asynchronous JavaScript and XML，即异步的 JavaScript 和 XML，所以 AJAX = 异步 Javascript + XML。

其中异步Javascript指的是 AJAX 的实现方式，XML 指的是响应的信息格式，不过现在一般使用 json 格式了。

AJAX 是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。

通过在后台与服务器进行少量数据交换，AJAX 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。

XMLHttpRequest 只是实现 Ajax 的一种方式。

为什么要有 AJAX

对于传统的网页（不使用 AJAX），如果需要更新网页上的部分内容，则必须重新加载整个网页。这对于浏览器和网络的压力是很大的。

有了 AJAX 之后，对于网页来说，可以只加载需要重新加载的内容。

AJAX 是如何实现的

XMLHttpRequest 是实现 AJAX 的一种常用方式。

使用了 AJAX 之后，浏览器页面可以通过js调用一个异步请求XMLHttpRequest，由这个请求向服务器请求资源，然后返回一个资源载体（可能是一个页面，也可能是一个xml或json），然后由js去处理这个信息。与此同时，页面是不会发生刷新行为的（也就是没有向服务器请求这个页面资源）。

AJAX 的优点

AJAX 可以在不重新加载整个页面的情况下，可以与服务器交换数据并更新部分网页内容。

AJAX 不需要任何浏览器插件，但需要用户允许 JavaScript 在浏览器上执行。

XMLHttpRequest

XMLHttpRequest 是 AJAX 的基础。

所有现代浏览器均支持 XMLHttpRequest 对象（IE5 和 IE6 使用 ActiveXObject）。

（4）响应状态判断

前面我们说过 AJAX 的请求是异步的，那么我们如何知道该异步请求被服务器处理完成呢？

首先XMLHttpRequest提供了readyState属性，用来表示请求处理到哪一步阶段了，请求状态包括下面 5 种：

  • 0：请求还未初始化
  • 1：服务器连接已建立
  • 2：请求已接收
  • 3：请求处理中
  • 4：请求处理完成

由此可以知道，当readyState为 4 的时候请求处理完成，我们就可以从XMLHttpRequest对象中获取 response 了。但是我们不能写一个死循环在其中一直判断readyState的取值，这样会导致CPU飙升。

一般对于这种异步请求完成的方式是通过回调函数来实现。

回调函数是一种以参数形式传递给另一个函数的函数。我们可以实现一个回调函数，在函数中判断readyState是否为4，当为4的时候说明请求处理完成，可以在后台更新页面了。

回调函数什么时候会被调用呢？对XMLHttpRequest来说，它提供了一个属性onreadystatechange可以用来指定回调函数，当readyState属性发生变化的时候会调用该回调函数。

最后一个需要注意的是：请求虽然处理完成了，但是可能会有处理失败的情况，我们可以通过XMLHttpRequest的 status 属性判断是否处理成功，只有成功之后才会重新加载网页内容。
