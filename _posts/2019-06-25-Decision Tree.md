决策树是一种基本的分类与回归方法。决策树模型呈树形结构，表示基于特征对实例进行分类的过程，可以认为是`if-then`规则的集合。其主要优点有模型可解释性好，分类速度快，对输入数据要求低。学习时，利用训练数据，根据损失函数最小化的原则建立决策树模型。预测时，对新的数据，利用决策树模型进行分类。决策树通常包含三个步骤：特征选择，决策树的生成和决策树的修剪。

本文介绍`ID3`，`C4.5`和`CART`算法，关于随机森林、`GBDT`、`Xgboost`和`LightGBM`将在后面介绍。

## 决策树基本算法

### 决策树生成算法

自顶向下的贪婪二分递归（选择当前最优属性，局部最优）


- 输入：训练集$D={(x_1,y_1),(x_2,y_2),\dots,(x_m,y_m)}$；属性集$A={a_1,a_2,\dots,a_d}$
- 过程：函数 `TreeGenerate(D,A)`
    - 生成节点`node`;
    - if D中样本全属于同一类别C `then`
        - 将node标记为C类叶结点；`return`
    - end if
    - if A为空或者D中样本在A上取值相同 then
        - 将node标记为叶结点，其类别标记为D中样本数最多的类；return
    - end if
    - 从A中选择最优划分属性$a_* $
    - for $a_* $的每一个取值 $a_* ^v$ do
        - 为node生成一个分支；令$D_v$表示D中在$a_* $上取值为$a_* ^v$的样本子集；
        - if $D_v$为空 then
            - 将分支节点标记为叶结点，其类别标记为D中样本最多的类；return
        - else
            - 以$TreeGenerate(D_v,A\\{a_*})$为分支节点
        - end if
    - end for

- 决策树生成算法停止的三种条件：
    1. 当前结点包含的样本属于同一类别，无需划分；
    2. 当前属性集为空，或所有样本在所有属性上取值相同，无法划分。此时将当前结点标记为叶结点，并将其类别设定为该结点所含样本最多的类别，实质是利用当前节点的后验分布；
    3. 当前结点包含的样本类别为空，不能划分。此时将当前结点标记为叶结点，并将其类别设定为父节点所含样本最多的类别，实质是把父节点的样本分布作为当前结点的先验分布。

### 最优属性选择

- 信息增益

假定样本集D中第k类样本所占的比例为$p_k$，则D的信息熵定义为：

$$H(D)=-\sum_{k=1}^Kp_klog(p_k)$$

假定离散属性a有V个可能的取值${a^1,a^2,\dots,a^V}$，若使用a对D进行划分，则会产生V个分支节点，其中第v个分支节点包含了D中所有在属性a上取值为$a^v$的样本，记为$D^v$。则使用属性a对D划分后所获得的信息增益为：

$$Gain(D,a)=H(D)-\sum_{v=1}^V\frac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}H(D^v)$$

信息增益越大，意味着使用属性a进行划分所获得的提升越大，故ID3算法基于信息增益来选择最优属性，即$a_*=\arg\max_{a\in A}Gain(D,a)$

实际上，信息增益偏好于取值较多的属性，为了减少这种偏好可能带来的不利影响，C4.5算法使用信息增益率来选择最优划分属性，使用属性a对D划分后所获得的信息增益率定义为：

$$Gain\_ratio(D,a)=\cfrac{Gain(D,a)}{H_a(D)}$$

其中${H_a(D)}$表示D按属性a划分的信息熵

$${H_a(D)}=-\sum_{v=1}^V\frac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}log(\frac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}})$$

信息率准则偏好于取值数目较少的属性，因此C4.5算法的实际属性选择策略是：先从候选划分属性中找出信息增益高于平均水平的属性，再从中选择增益率最高的。

- 基尼指数

基尼值：从D中随机抽取两个样本，类别不一致的概率，定义为：

$$Gini(D)=\sum_{k=1}^K\sum_{k'\neq k}p_kp_{k'}=1-\sum_{k=1}^Kp_k^2$$

使用属性a对D划分后的基尼指数定义为：

$$Gini\_index(D,a)=\sum_{v=1}^V\cfrac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}Gini(D^v)=\cfrac{\begin{vmatrix}D_1\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}Gini(D_1)+\cfrac{\begin{vmatrix}D_2\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}Gini(D_2)$$

CART是二叉树，对于类别型离散特征，根据特征a是否取某一可能值$a^v$将D划分为$D_1$和$D_2$两部分（注意对于多个取值的类别特征，每次划分时只考虑一个可能的取值，有点类似于`one vs rest`）。对于数值型连续特征，将样本取值从小到大排列，依次将所有相邻取值的均值作为候选划分点，根据指标选择一个最佳划分点进行划分，注意数值型连续特征后续还可以进行划分。CART算法选择划分后基尼指数最小的属性，$a_*=\arg\max_{a \in A}Gini_index(D,a)$

3.不同形态的树模型，差异在哪里，如何改进？

ID3在决策树的各级节点上，使用信息增益方法作为属性的选择标准，来确定生成每个节点时所采用的属性。C4.5的改进是使用信息增益率来选择节点属性。克服了ID3存在的不足：ID3只适用于离散的描述属性，C4.5可以处理离散的描述属性和连续的描述属性。CART是一种十分有效的非参数分类与回归方法，通过构建树、修剪树、评估树来构建一个二叉树。当终结点是连续变量时，该树为回归树；当终结点是分类变量时，该树是分类树。

|-|-|-|-|-|-|-|-|
|ID3|最大信息增益|离散型|不支持|不支持|多叉树|离散特征不可重复使用，不支持连续特征|分类|
|C4.5|最大信息增益比|离散型，连续型|支持|支持|多叉树，每个特征不可重复使用|离散特征不可重复使用，连续特征可重复使用|分类|

4.连续值如何处理

ID3算法不支持对连续特征进行处理，C4.5和CART采用二分法对连续属性进行处理。对其所有取值，按从小到大的顺序排列，依次取相邻两个的均值$\frac{x_i+x_{i+1}}{2}$作为划分点，一共选出`m-1`个划分点集合。

对C4.5算法，计算`m-1`个划分点的信息增益比，同时计算其他特征生成的信息增益比，从中选择最大的信息增益比对应的特征以及划分点。对于CART算法，计算`m-1`个划分点的基尼系数，同时计算所有其他特征生成的基尼系数，从中选择最小的基尼系数对应的特征及其二分点。

对于连续特征，每次会有`m-1`个候选划分点，如果当前结点选择该特征的某个二分点作为划分点，在之后的划分节点选择中，还可以选择该特征的其他划分点。

5.缺失值如何处理

针对缺失值，需要解决以下两个问题：

- 在有缺失值的情况下，如何确定划分特征？
- 当划分特征已经确定，若样本在该特征上的值缺失，该样本应该划分到哪个节点？

C4.5和CART都解决了这两个问题。对于一个有缺失特征值的特征$a$,对每个样本设置一个权重（初始化为1），根据该特征是否缺失将样本划分为两部分，令$\tilde{D}$表示D中在属性a上没有缺失值的样本子集，用$\rho$表示无缺失样本子集占所有数据集的比例。计算特征$a$在无缺失样本子集$\tilde{D}$上的信息增益比或者基尼系数，然后乘以权重$\rho$得到最终结果。

对于第二个问题，如果划分节点为特征$a$，某样本在该特征下的值是缺失的，则将该样本划分到所有子结点中，且样本权重根据样本子集$\tilde{D}$中特征$a$在不同取值下所占样本的比例来分配，即样本权重在与属性值$a^v$对应的子结点中调整为$\frac{\begin{vmatrix}\widetilde{D}^v\end{vmatrix}}{\begin{vmatrix}\widetilde{D}\end{vmatrix}}$,实质就是让同一个样本以不同的概率划入到不同的子结点中去。

xgboost、LightGBM自有一套处理缺失值的方法

9.决策树如何做回归

CART基于基尼指数最小化准则生成二叉分类树，基于平方误差最小化准则生成二叉回归树。

最小二乘回归树生成算法：

- 输入：训练数据集$D=\begin{Bmatrix}(x_1,y_1),(x_2,y_2),\dots,(x_N,y_n)\end{Bmatrix}$
- 输出：回归树$f(x)$
- 在D所在的输入空间中，递归地将每个区域划分为两个子区域并决定每个子区域上的输出值，构建二叉决策树：
- （1）选择最优切分变量$j$和切分点$s$，求解

$$\min_{j,s}[\min_{c_1}\sum_{x_i\in R_1(j,s)}(y_i-c_1)^2+\min_{c_2}\sum_{x_i\in R_2(j,s)}(y_i-c_2)^2]$$

- 遍历变量$j$，对固定的切分变量$j$扫描切分点$s,选择使上式达到最小值的对$(j,s)$
- （2）用选定的对$(j,s)$划分区域并决定相应的输出值：

$$R_1(j,s)=\begin{Bmatrix}x\mid x^{(j)}\leq s\end{Bmatrix},R_2(j,s)=\begin{Bmatrix}x\mid x^{(j)}> s\end{Bmatrix}$$

$$\hat{c}_m=\frac{1}{N_m}\sum_{x_i\in R_m(j,s)}y_i,x\in R_m,m=1,2$$

- （3）继续对两个子区域$R_1$和$R_2$调用步骤（1）和（2），直至满足停止条件
- （4）将输入空间划分为$M$个区域$R_1,R_2,\dots,R_M$，生成决策树：

$$f(x)=\sum_{m=1}^M\hat c{_m}I(x\in R_m)$$

这里的停止条件可以为：总误差小于设定的阈值时；区域划分次数达到要求的次数时

11.决策树的优缺点

- 优点
    - 非黑盒，可解释
    - 轻松去除无关的属性（`Gain`为0或很小）
    - 测试很快，`O(depth)`
- 缺点
    - 只能线性分割数据
    - 贪婪算法，选择局部最优划分属性，可能找不到最好的树
