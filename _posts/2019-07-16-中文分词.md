中文分词根据实现原理和特点，主要分为基于词典匹配的分词方法和基于统计的分词方法。

## 分词算法设计中的几个基本原则：

1、颗粒度越大越好：用于进行语义分析的文本分词，要求分词结果的颗粒度越大，即单词的字数越多，所能表示的含义越确切，如：“公安局长”可以分为“公安 局长”、“公安局 长”、“公安局长”都算对，但是要用于语义分析，则“公安局长”的分词结果最好（当然前提是所使用的词典中有这个词）

2、切分结果中非词典词越少越好，单字字典词数越少越好，这里的“非词典词”就是不包含在词典中的单字，而“单字字典词”指的是可以独立运用的单字，如“的”、“了”、“和”、“你”、“我”、“他”。例如：“技术和服务”，可以分为“技术 和服 务”以及“技术 和 服务”，但“务”字无法独立成词（即词典中没有），但“和”字可以单独成词（词典中要包含），因此“技术 和服 务”有1个非词典词，而“技术 和 服务”有0个非词典词，因此选用后者。

3、总体词数越少越好，在相同字数的情况下，总词数越少，说明语义单元越少，那么相对的单个语义单元的权重会越大，因此准确性会越高。

## 汉语分词存在的困难
### 分词规范
不同人对“词”的理解不同，划分方式也不同。
### 歧义词划分
歧义字段在汉语文本中普遍存在。梁南元1987年最早对歧义字段进行了比较系统的考察，定义了交集型切分歧义和组合型切分歧义两种类型。
- 交集型切分歧义：对于汉字串AJB，如果AJ、JB同时为词，此时汉字串J称为交集串，汉字串AJB称为交集型切分歧义。交集型切分歧义在汉语文本中普遍存在，如“组合成”、“大学生”、“研究生物”、“从小学起”，“为人民工作”，“中国产品质量”，“部分居民生活水平”等。进一步地，将交集串的集合称为交集串链，交集串的个数称为链长。如“为人民工作”的交集串链为{人，民，工}，链长为3；“部分居民生活水平”的交集串链为{分，居，民，生，活，水}，链长为6。
- 组合型切分歧义：对于汉字串AB，如果满足A,B,AB同时为词，称AB为组合型切分歧义。如“起身”（他/站/起/身/来，他/明天/起身/去/北京）,“将来”，“现在”，“才能”，“学生会”。
- 多义组合型切分歧义：对于汉字串AB，如果(1)AB,A,B同时成词；(2)文本中至少存在一个上下文语境C，在C的约束下，A,B在语法和语义上都成立，则汉字串AB称为多义组合型切分歧义。

刘开瑛在2000年的统计结果显示，在交集型切分歧义字段中，绝大多数为链长为1和2的歧义字段。

交集型字段可能包含组合型字段，即交集字段的长度大于组合型字段的长度，如:
```
（1）这篇文章写得太平淡了
（2）这墙抹的太平了
（3）即使太平时期也不应该放松警惕
```
“太平淡”是交集型字段，“太平”是组合型字段，即交集型字段包含组合型字段。处理这类歧义字段必须首先处理交集型字段，如果匹配不成功，在短语层面再按组合型字段处理。
### 未登录词识别
未登录词包含两方面的含义：一是现有的词典中没有的词（对应基于词典匹配的方法），二是训练语料中未出现的词（对应基于大规模训练语料的统计方法）。词典中未出现的词包括新出现的网络流行语、词典未收录的人名地名、音译词。第二种含义下未登录词又称为OOV，即out of vocabulary 训练集之外的词。

对于大规模真实文本来说，研究发现，未登录词对于分词精度的影响远远超过了歧义词。基于词典的方式较难解决未登录词的问题，简单的case可以通过加词典解决，但是随着字典的增大，可能会引入新的bad case，并且系统的运算复杂度也会增加。

## 基于词典匹配的分词方法
基于词典匹配的分词方法，也叫做字符串匹配分词方法。该方法按照一定的策略将待匹配的字符串和已经建立好的词典中的词进行匹配，若在词典中找到该词条，说明匹配成功，将该词条作为一个分词结果。基于不同的策略，可以分为正向最大匹配法、逆向最大匹配法和双向匹配分词法。
### 正向最大匹配法
#### 算法
首先设定最大匹配字符k，可设置为词典中存在的最长词的字符数，然后从左到右取k个字符，查找是否存在于词典中，存在则将这k个字符作为一个分词，重复取下k个字符；不存在则去除最右边一个字符（将字符长度减一），查找k-1个字符是否存在于词典中。直到字符串匹配完成。
#### 实现
#### 例子
待匹配字符串：南京市长江大桥

词典：南京、南京市、市长、长江、长江大桥、江大桥

匹配过程：k=4, 南京市长（不匹配），南京市（匹配），长江大桥（匹配），结果：南京市/长江大桥
### 逆向最大匹配法
#### 算法
首先设定最大匹配字符k，可设置为词典中存在的最长词的字符数，然后从右到左取k个字符，查找是否存在于词典中，存在则将这k个字符作为一个分词，重复取下k个字符；不存在则去除最左边一个字符（将字符长度减一），查找k-1个字符是否存在于词典中。直到字符串匹配完成。
#### 实现
#### 例子
待匹配字符串：南京市长江大桥

词典：南京、南京市、市长、长江、长江大桥、江大桥

匹配过程：k=4, 长江大桥（匹配），南京市（匹配），结果：南京市/长江大桥

### 双向最大匹配法
正向最大匹配法和逆向最大匹配法，都有其局限性，因此有人又提出了双向最大匹配法。即，前向和后向两种算法都切一遍，然后根据大颗粒度词越多越好，非词典词和单字词越少越好的原则，选取其中一种分词结果输出。

## 基于统计的方法
基于统计的机器学习算法主要有HMM、CRF、SVM、深度学习等。作为一个典型的标注任务，其基本思路是对汉字进行标注训练。
### 全切分路径选择方法

全切分方法就是将所有可能的切分组合全部列出来，并从中选择最佳的一条切分路径。关于路径的选择方式，一般有N-最短路径方法，基于词的n元语法模型方法等。

### N-最短路径方法

考虑到汉语分词中存在歧义消除和未登录词识别两个问题，有专家将分词过程分为两个阶段：首先采用切分算法对句子进行初步切分得到一个相对最好的粗分结果，然后，再进行歧义消除和未登录词识别。

张华平在2002年提出了基于N-最短路径方法的汉语词语粗分模型。基本思想为：根据词典，找出字符串中所有可能的词，将所有的切分结果组成有向无环图。其中每个词对应一条有向边，从词的第一个字符指向最后一个字符，并赋予相应的边长（权值）。针对该切分图，找出从起点到终点长度最小的N条路径集合作为粗分的结果集。

根据有向边的权重，该方法分为所有词权重都相等的非统计粗分模型和统计粗分模型。
- 统计粗分模型

假设字符串为C，分词结果为W，则此方法转变为求最大的N个已知C求W的条件概率对应的分词结果W。根据贝叶斯公式，等价于求C和W的联合概率最大的N个W，由于由分词结果得到字符串的概率为1，等价于求P(W)最大的N个划分结果。
### 基于词的n元语法模型分词方法
生成式模型早期的统计分词方法以此为基本模型，配合其他未登录词识别模块进行扩展。基本思想是：首先根据词典对句子进行简单匹配，找出所有可能的词典词，然后将它们和所有单个字作为节点，构造n元切分词图。图中的节点表示可能的词候选，边表示路径，边上的n元概率表示代价，最后利用相关搜索算法（如维特比）从图中找到代价最小的路径作为最后的分词结果。

基于词的n元语法模型可以看作是N-最短路径方法的一种优化，不同的是，根据N-元语法模型，路径构成时会考虑词的上下文关系，根据语料库的统计结果，找出构成句子最大模型概率。一般情况下，使用unigram和bigram的n元语法模型的情况较多。

### 由字构词（基于字标注）的汉语分词方法
薛念文在《Combining Classifiers for Chinese Word Segmentation》中首次提出对每个字符进行标注，通过机器学习算法训练分类器进行分词，在论文《Chinese word segmentation as character tagging》中较为详细地阐述了基于字标注的分词法。

将分词过程看做字的分类问题，认为每个字在构造一个特定的词语时都占据着一个特定的构词位置（词位）。假如规定每个字只有4个词位：词首B、词中M、词尾E和单独成词S。
```
上海计划到本世纪末实现人均国内生产总值五千美元。
上/B海/E计/B划/E到/S本/S世/B纪/E末/S实/B现/E人/B均/E国/B内/E生/B产/E总/B值/E五/B千/M美/M元/E。/S
```
将分词结果表示为字标注形式之后，分词问题就变味了序列标注问题。通常使用基于字的判别式模型时需要在当前字的上下文开一个w个字的窗口（一般取5），在这个窗口里抽取分词相关的特征。
```
北京奥运会
(a)一元语法：北、京、奥、运、会
(b)二元语法：北京、京奥、奥运、运会
(c)前一个字+后一个字：京运
(d)与数字、标点符号和英文字符相关的词
```
得到这些特征之后，利用常见的判别式模型，如最大熵、条件随机场、支持向量机、感知机等进行参数训练，然后利用解码算法找到最优的切分结果。

由字构词的分词方法可以平衡地看待词典词和未登录词的识别问题，文本中的词典词和未登录词都是用统一的字标注过程实现的，分词过程成为字重组的简单过程。在学习构架上，既不用专门强调词典词信息，也不用专门设计特定的未登录词识别模块，大大简化了分词系统的设计。
