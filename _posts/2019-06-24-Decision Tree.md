决策树是一种基本的分类与回归方法。决策树模型呈树形结构，表示基于特征对实例进行分类的过程，可以认为是`if-then`规则的集合。其主要优点有模型可解释性好，分类速度快，对输入数据要求低。学习时，利用训练数据，根据损失函数最小化的原则建立决策树模型。预测时，对新的数据，利用决策树模型进行分类。决策树通常包含三个步骤：特征选择，决策树的生成和决策树的修剪。

本文首先介绍ID3，C4.5和CART算法，随后介绍随机森林和GBDT，最后介绍Xgboost和LightGBM。

## 决策树基本算法

### 决策树生成算法

自顶向下的贪婪二分递归（选择当前最优属性，局部最优）


输入：训练集$D={(x_1,y_1),(x_2,y_2),\dots,(x_m,y_m)}$；属性集$A={a_1,a_2,\dots,a_d}$

过程：函数 `TreeGenerate(D,A)`

    生成节点node;

    if D中样本全属于同一类别C then

        将node标记为C类叶结点；return

    end if

    if A为空或者D中样本在A上取值相同 then

        将node标记为叶结点，其类别标记为D中样本数最多的类；return

    end if

    从A中选择最优划分属性$a_*$

    for $a_*$的每一个取值 $a_*^v$ do

        为node生成一个分支；令$D_v$表示D中在$a_*$上取值为$a_*^v$的样本子集；

        if $D_v$为空 then

            将分支节点标记为叶结点，其类别标记为D中样本最多的类；return

        else

            以TreeGenerate($D_v$,$A\{a_*}$)为分支节点

        end if

    end for


- 决策树生成算法停止的三种条件：
    1. 当前结点包含的样本属于同一类别，无需划分；
    2. 当前属性集为空，或所有样本在所有属性上取值相同，无法划分。此时将当前结点标记为叶结点，并将其类别设定为该结点所含样本最多的类别，实质是利用当前节点的后验分布；
    3. 当前结点包含的样本类别为空，不能划分。此时将当前结点标记为叶结点，并将其类别设定为父节点所含样本最多的类别，实质是把父节点的样本分布作为当前结点的先验分布。

### 最优属性选择

- 信息增益

假定样本集D中第k类样本所占的比例为$p_k$，则D的信息熵定义为：

$$H(D)=-\sum_{k=1}^Kp_klog(p_k)$$

假定离散属性a有V个可能的取值${a^1,a^2,\dots,a^V}$，若使用a对D进行划分，则会产生V个分支节点，其中第v个分支节点包含了D中所有在属性a上取值为$a^v$的样本，记为$D^v$。则使用属性a对D划分后所获得的信息增益为：

$$Gain(D,a)=H(D)-\sum_{v=1}^V\frac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}H(D^v)$$

信息增益越大，意味着使用属性a进行划分所获得的提升越大，故ID3算法基于信息增益来选择最优属性，即$a_*=\arg\max_{a\in A}Gain(D,a)$

实际上，信息增益偏好于取值较多的属性，为了减少这种偏好可能带来的不利影响，C4.5算法使用信息增益率来选择最优划分属性，使用属性a对D划分后所获得的信息增益率定义为：

$$Gain\_ratio(D,a)=\cfrac{Gain(D,a)}{H_a(D)}$$

其中${H_a(D)}$表示D按属性a划分的信息熵

$${H_a(D)}=-\sum_{v=1}^V\frac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}log(\frac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}})$$

信息率准则偏好于取值数目较少的属性，因此C4.5算法的实际属性选择策略是：先从候选划分属性中找出信息增益高于平均水平的属性，再从中选择增益率最高的。

- 基尼指数

基尼值：从D中随机抽取两个样本，类别不一致的概率，定义为：

$$Gini(D)=\sum_{k=1}^K\sum_{k'\neq k}p_kp_{k'}=1-\sum_{k=1}^Kp_k^2$$

使用属性a对D划分后的基尼指数定义为：

$$Gini\_index(D,a)=\sum_{v=1}^V\cfrac{\begin{vmatrix}D^v\end{vmatrix}}{\begin{vmatrix}D\end{vmatrix}}Gini(D^v)$$

CART算法选择划分后基尼指数最小的属性，$a_*=\arg\max_{a \in A}Gini\_index(D,a)$

### 面试问题

1.树模型的形态（假设函数）

    - ID3和C4.5针对离散属性，是多叉树，用于分类任务
    - CART是二叉树，用于分类和回归任务，可以处理离散属性和连续属性
    - CART如何处理多个取值的类别属性：将每个取值作为一个特征

2.树模型的生成过程（如何找到最佳属性，什么时候停止生长）

3.不同形态的树模型，差异在哪里，如何改进？

4.连续值如何处理

C4.5中采用二分法对连续属性进行处理，对其所有取值，按从小到大的顺序排列，依次取相邻两个的均值作为划分点，一共选出n-1个划分点集合。计算每个划分点划分后的信息增益，选择信息增益最大的划分点作为对应属性的最优划分点。

5.缺失值如何处理
- 西瓜书的做法：(p85-88)
    - 在划分属性选择阶段，只考虑当前属性无缺失样本
    - 在样本划分阶段，让同一个样本以不同的概率划入到不同的子结点中
- xgboost、LightGBM自有一套处理缺失值的方法

6.过拟合的产生原因，缓解方法
- 产生原因
    - 叶子节点上的样本数很少
- 缓解方法
    - 剪枝
    - 限制树深
    - 限制叶子结点的样本数
    - 限制节点划分的最小信息增益
    - 集成
    - 加正则项

7.DT中特征重要性如何计算

- 计算属性划分带来的信息增益
- 计算属性的划分次数

8.DT生长时，属性可以多次使用吗？

- ID3和C4.5
    - 数值型连续属性可以多次划分
    - 类别型离散属性不能多次划分
- CART
    - 数值型连续属性可以多次划分
    - 类别型离散属性在不同分支中可以多次使用
