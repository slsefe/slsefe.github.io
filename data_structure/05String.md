## 串的定义(string)
- 串(string):由零个或多个字符组成的有限序列,又叫字符串.
- 记法:一般记为`s="a1a2......an"(n>=0)`,其中s是串的名称,用双引号括起来的字符序列是串的值.
- 字符串的字符数目n称为字符串的长度,零个字符的串称为空串.
- 空格串:只包含空格的串
- 子串和主串:串中任意个数的连续字符组成的子序列称为该串的子串,包含子串的串称为主串.
## 串的比较
- 串的比较通过比较组成串的字符之间的编码来进行,字符的编码指的是字符在相应字符集中的序号.
- ASCII编码:用8位二进制数表示一个字符,一共可以表示256个字符
- Unicode编码:用16位二进制数表示一个字符,一共可以表示6万多个字符

## 朴素的模式匹配算法
- 定义:**子串的定位操作通常称为串的模式匹配**
- 操作:对主串的每一个字符作为子串开头,与要匹配的字符串进行匹配.直到匹配成功或全部遍历完成为止.
- 实现:
```
def index(s1, s2):
    # s1为主串,s2为子串
    i, j = 0, 0
    while i < len(s1) - len(s2) + 1 and j < len(s2):
        if s1[i] == s2[j]:
            i += 1
            j += 1
        else:
            # i-j+1表示当前匹配的次数,也是下一次匹配时的第一个位置
            i = i - j + 1
            j = 0
    if j == len(s2):
        # 若匹配成功,j最后一次迭代后会等于len(s2),返回当前匹配的第一个位置
        return i - len(s2)
    else:
        return -1
```
- 性能分析
设n为主串长度,m为子串长度
    - 最好情况:第一次就匹配成功,时间复杂度为O(m),即O(1)
    - 次好情况:对于所有匹配不成功的情况,第一位均不匹配.第一次匹配成功时间复杂度为O(1),最后一次匹配成功时间复杂度为O(n+m),平均时间复杂度为O(n+m)
    - 最坏情况:对于所有匹配不成功的情况,均为最后一位不匹配.最后一次匹配成功的时间复杂度为O((n-m+1)*m)
